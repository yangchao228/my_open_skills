## 复现路径

1. 打开参与 skill 的 `SKILL.md` 和最小示例
2. 从公开或已授权源文提取可复现的输入
3. 记录运行环境、日期、实际输出和待核字段
4. 对照每个 skill 的输出合同检查结果
5. 将问题反馈到仓库 Issue，并附最小可复现输入

## 发布检查

- [ ] 仓库链接和 skill 路径有效
- [ ] 输入已脱敏
- [ ] 输出来自实际运行
- [ ] 所有效果和热度声明有来源
- [ ] 热门标签包含当前来源和检查时间；未核验时明确标为 `unverified`
- [ ] 时效信息标明日期
- [ ] 公众号和知乎分别完成插图判断
- [ ] `publish_ready` 卡片包已经进入 `cards-to-images`，没有停在文案或提示词
- [ ] 卡片图片真实存在，数量、顺序、尺寸和 manifest 一致
- [ ] 每张卡片已经检查文字、裁切、留白、顺序和视觉一致性
- [ ] 人工确认状态如实记录；未确认时没有写成已发布
- [ ] 本地图片已经插入正式 Markdown 后才进入 `md-img-r2`
- [ ] R2 上传、Markdown 改写和平台发布分别保留人工确认
- [ ] 两个平台没有直接复制同一篇文案
- [ ] 每个平台只有一个 CTA
- [ ] 没有超出 skill 已公开边界
