# 为什么我给小红书单独做了一个选题 Skill？

> 我亲手打造的 Skills · 01

现在让 AI 生成标题，已经简单到几乎没有门槛。

输入一个主题，再要求“给我 20 个吸睛标题”，很快就能得到一串结果：

- AI 时代，每个人都应该拥有自己的 Skill
- 我用一个 Skill，解决了大部分选题焦虑
- 普通人如何靠 Skills 实现效率翻倍

这些标题读起来很顺，也可以被替换到许多主题和平台。真正需要作者完成的判断还留在原地：

- 这个主题适合小红书吗？
- 谁会在什么场景下点开？
- 读者看完能带走什么？
- 作者有没有真实材料支撑？
- 这个选题值得投入时间吗？

这也是我开始公开“我亲手打造的 Skills”系列时，想先讲清楚的一件事：一个能长期复用的 Skill，价值来自一套可以重复执行的判断过程。

## 我让这个 Skill 给自己做了一次选题

我的公开仓库目前有 15 个 `ready` skills，其中一个叫 `xiaohongshu-topic-generator`。它负责把宽泛主题转成更适合小红书的选题、卡片框架和发布前风险检查。

这次，我把“我亲手打造的 Skills”这个系列交给了它。

公开输入包括：

- 目标读者是正在使用 AI，又想沉淀个人工作流的人
- 作者素材来自真实构建和维护公开 Skills 的经历
- 输出要适合 3:4 图文卡片
- 不使用未经验证的“爆款”“最近很火”“效率翻倍”等承诺
- 每个候选题都要说明读者场景、卡片框架、评分和证据风险

本轮没有提供近期平台数据，所以所有候选题统一标注“时效未验证”。

实际得到的方向包括：

- 输入一句主题，先筛掉不值得投入的选题
- 给选题 Skill 加五项评分，减少“看起来不错”的候选
- 没有当前来源时，为什么必须写“时效未验证”
- 把常用 Prompt 整理成公开 Skill，需要补哪些结构
- 哪些创作者适合这个 Skill，哪些需求不在它的能力范围内
- 一个私人 workflow 公开前要经过哪些检查

输出里还有目标读者、第一句话、卡片顺序、读者收获、视觉建议、风险提示，以及“优先做 / 可备选 / 不建议投入”的结论。

标题只是其中一个字段。

## 设计判断一：平台会从一开始影响选题

小红书和知乎需要的内容判断不同。

这个 Skill 会先检查一个选题是否具备具体场景、明确痛点、实用方法、首屏钩子、卡片结构、特定人群、收藏价值和可信作者材料。通常至少满足其中四项，才值得继续投入。

例如“为什么 Skill 是 AI 时代的重要能力”具备讨论空间，放在知乎可以展开概念、判断和争议。直接放到小红书，它容易停在抽象观点。

换成“我把反复使用的选题流程整理成一个 Skill，里面保留了哪五个判断”，场景、方法和视觉结构会清晰很多。

平台会影响选题从哪里进入、用什么结构展开、读者最后能保存什么。把平台适配留到发布前，前面的内容结构往往已经偏了。

## 设计判断二：主动淘汰候选，减少无效投入

万能生成器很擅长增加候选数量。

真实创作中，我通常不缺标题，真正稀缺的是时间和注意力。十个看起来都不错的标题，常常意味着作者还要重新完成一次筛选。

`xiaohongshu-topic-generator` 会主动拉开内容类型：实用指南、个人经历、对比、决策辅助、风险边界和迷你案例。这样可以避免五个候选只是同一句话的改写。

随后，它会按照五个维度评分：

1. 受众相关性
2. 平台适配度
3. 钩子强度
4. 实用价值
5. 作者可写性

每项 1-5 分，总分 25 分。这个分数只用于内部排序，不代表小红书平台评分，也不预测流量。

更重要的是，它允许给出“不建议投入”。

只有宏大观点、缺少用户场景的方向会被降级；和主题关系很弱的热点不会被强行嫁接；作者没有真实材料支撑的结果型标题也不应该进入优先列表。

一个稳定创作系统要能帮助人缩小选择范围，同时把最终取舍留给作者。

## 设计判断三：把证据边界放进选题阶段

内容生成很容易在选题阶段说得过头。

这个 Skill 要求每个候选题都附带“时效判断”和“风险或证据检查”。用户要求追近期热点时，需要当前来源支持；无法获得近期信号，就明确标注“时效未验证”。涉及结果、数据、排名和趋势的说法，也要列出证据缺口。

这条规则会减少一些看起来更有冲击力的标题，也能减少后续返工。

我可以公开它如何工作、为什么这样设计、当前解决了哪些问题。我不会把一次本地演示包装成普遍成效，也不会声称它已经获得平台验证。

## 它目前做不到什么

`xiaohongshu-topic-generator` 提供的是一套选题判断框架，它不会替代账号经营者的最终判断。

- 它不知道你的真实读者最关心什么，除非你提供受众和历史材料
- 没有近期来源时，它不能确认某个话题正在变热
- 默认输出停在选题资产，不继续生成完整正文
- 它不会设计固定品牌视觉，也不会执行账号发布
- 它不能承诺流量、涨粉或节省多少时间

同一个主题进入知乎后，还要重新检查推理空间、争议点、作者立场和长期讨论价值。直接搬运标题，只完成了格式迁移，平台判断仍然缺失。

这些限制会继续保留在 Skill 里。边界越明确，别人越容易安全地复用和调整。

## 如何复现这次案例

公开 Skill 在这里：

[GitHub：xiaohongshu-topic-generator](https://github.com/yangchao228/my_open_skills/tree/main/skills/content/xiaohongshu-topic-generator)

当前可验证的使用方式，是克隆仓库后，把这个 Skill 目录复制或链接到支持 Agent Skills 的运行环境。你也可以先阅读 `SKILL.md`、最小输入示例和预期输出说明。

这次使用的完整公开输入、六个候选题和验收条件，也已经整理成 [Case Pack](case-pack.md)。

可以从这条最小指令开始：

> 请围绕“我最近亲手做的一个 AI 工作流”生成 6 个小红书图文选题。目标读者是想把日常经验沉淀成可复用资产的人。不要追未经验证的热点，每个选题给出标题、首句、卡片框架、评分和风险检查。

“我亲手打造的 Skills”会继续公开这些真实设计过程：它解决什么问题、我做了哪些判断、哪里仍有限制，以及别人怎样把它改成自己的版本。
